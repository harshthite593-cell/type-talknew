WHAT TO BUILD: SAVE & REUSE PHRASES FEATURE
═══════════════════════════════════════════════════════════════════

PURPOSE:
Users will type the same phrases repeatedly ("I want water", 
"Thank you", "I love you"). Let them save these as quick-access 
buttons so they don't have to retype every time.

IMPLEMENTATION SPECIFICATION:

1. ADD A SMALL "SAVE" BUTTON
   - Location: Next to the existing Enter/Speak button
   - Icon: Bookmark or heart icon (💾 or ⭐)
   - Size: 48dp × 48dp (smaller than main Speak button)
   - Color: Teal (#4ECDC4) background, white icon
   - Behavior: When tapped, saves whatever is currently in the 
     input text field

2. CREATE A "SAVED PHRASES" SCREEN
   - Add a new screen accessible from a menu icon or "My Phrases" 
     button somewhere visible on home screen
   - Screen shows list of all saved phrases
   - Each saved phrase card has:
     * The phrase text (large, readable, 18sp)
     * A "🔊 Speak" button (taps to speak instantly via TTS)
     * A "🗑️ Delete" button (removes the phrase)
   - Empty state message: "No saved phrases yet. Save your 
     favorites by tapping the bookmark icon!"

3. DATA STORAGE
   - Use simple localStorage (if web app) or AsyncStorage 
     (if React Native) or SQLite (if native)
   - Store as JSON array of objects:
     {
       id: timestamp,
       text: "the phrase",
       savedAt: ISO date string
     }
   - Persist across app restarts

4. SAVE LOGIC
   - When user taps Save button:
     a) Check if input field has text (not empty)
     b) Check if phrase already exists (prevent duplicates)
     c) Add to saved phrases list
     d) Show brief toast/snackbar: "Phrase saved! ✅"
     e) Keep text in input field (don't clear it)
   
   - If input is empty, show: "Type something first to save"
   - If duplicate, show: "This phrase is already saved"

5. NAVIGATION
   - Add a "My Phrases" link/button on home screen
   - OR add a side menu with "Saved Phrases" option
   - Keep it accessible but not interfering with main typing area

6. UI/UX REQUIREMENTS
   - Maintain existing design language (colors, fonts)
   - Smooth transitions between home and saved phrases screen
   - Back button to return to home from saved phrases screen
   - All buttons must be at least 48dp for accessibility
